TERMINAL.


> 01. Installing StrongSwan.

a. Update local packages: sudo apt update

b. Install StrongSwan + additional packages: sudo apt install strongswan strongswan-pki libcharon-extra-plugins libcharon-extauth-plugins libstrongswan-extra-plugins


> 02. Creating a Certificate Authority.

a. Create directories to store assets: mkdir -p ~/pki/{cacerts,certs,private}

b. Secure permissions to hide from other users: chmod 700 ~/pki

c. tpm 2 error: sudo apt-get install libtss2-tcti-tabrmd-dev

d. Generate root key: pki --gen --type rsa --size 4096 --outform pem > ~/pki/private/ca-key.pem

e. Create root CA using root key to sign root certificate: pki --self --ca --lifetime 3650 --in ~/pki/private/ca-key.pem \ --type rsa --dn "CN=VPN root CA" --outform pem > ~/pki/cacerts/ca-cert.pem


> 03. Generating a Certificate for the VPN Server.

a. Create a private key for the VPN server: pki --gen --type rsa --size 4096 --outform pem > ~/pki/private/server-key.pem

b. Create and sign the VPN server certificate, with the CA's key created in the previous step: pki --pub --in ~/pki/private/server-key.pem --type rsa \ | pki --issue --lifetime 1825 \ --cacert ~/pki/cacerts/ca-cert.pem \ --cakey ~/pki/private/ca-key.pem \ --dn "CN=ec2-18-218-127-65.us-east-2.compute.amazonaws.com" --san ec2-18-218-127-65.us-east-2.compute.amazonaws.com \ --flag serverAuth --flag ikeIntermediate --outform pem \ > ~/pki/certs/server-cert.pem

c. Move files we created to the appropriate directory: sudo cp -r ~/pki/* /etc/ipsec.d/

> 04. Configuring StrongSwan.

a. Backup original config file: sudo mv /etc/ipsec.conf{,.original}

b. Create and open a new blank configuration file: sudo nano /etc/ipsec.conf

c. Add these lines: config setup charondebug="ike 1, knl 1, cfg 0" uniqueids=no conn ikev2-vpn auto=add compress=no type=tunnel keyexchange=ikev2 fragmentation=yes forceencaps=yes dpdaction=clear dpddelay=300s rekey=no left=%any leftid=@ec2-18-218-127-65.us-east-2.compute.amazonaws.com leftcert=server-cert.pem leftsendcert=always leftsubnet=0.0.0.0/0 right=%any rightid=%any rightauth=eap-mschapv2 rightsourceip=10.10.10.0/24 rightdns=8.8.8.8,8.8.4.4 rightsendcert=never eap_identity=%identity ike=chacha20poly1305-sha512-curve25519-prfsha512,aes256gcm16-sha384-prfsha384- ecp384,aes256-sha1-modp1024,aes128-sha1-modp1024,3des-sha1-modp1024! esp=chacha20poly1305-sha512,aes256gcm16-ecp384,aes256-sha256,aes256-sha1,3des-sha1!

> 05. Configuring VPN Authentication

a. Open the secretes file: sudo nano /etc/ipsec.secrets

b. Tell StrongSwan where to find private key and how to parse:: RSA "server-key.pem"

c. Define user credentials: username : EAP "password"

d. Restart VPN service: sudo systemctl restart strongswan-starter
